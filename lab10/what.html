What is it?

Definition

Web Crypto API is a powerful tool consisting of best practices like encryption, signing, and hashing used to secure your web applications. It is a low-level interface made to secure web apps directly within the web browser. These best practices are a set of methods called “cryptographic primitives.” These methods aid in developing and implementing secure patterns. It also adds integrity, one of the three main goals of the CIA triad, to a system.

Purpose

Web Crypto API was created because there was a lack of native cryptographic functionality in JavaScript, requiring third-party libraries with potential security risks. Another reason it was created is to provide a baseline across all browsers and web pages for how to secure it and manage said security risks. Basically, it was made in order to standardize securing and encrypting web applications. The WCA was also made in order to provide users with client-side security which is way safer than server-side security.

Who Made It and When

Web Crypto API was made by the Web Cryptography Working Group (WCWG), which is a part of the World Wide Web Consortium (W3C). The specification for the Web Crypto API was published on January 26, 2017.
